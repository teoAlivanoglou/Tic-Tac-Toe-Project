># Tic-Tac-Toe
>Τελική εργασία - Παιχνίδι Τρίλιζα με γραφικό περιβάλλον (Tkinter).

## Προτάσεις
1. Χρήση [vscode](https://code.visualstudio.com/).
2. Χρήση [GitHub](https://github.com/).
3. Χρήση των recommended extensions: <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>P</kbd> -> "**Extensions: Show Recommended Extensions**".
4. Ότι γράφουμε, το γράφουμε στο δικό μας branch, όχι στο main (για αποφυγή conflicts).
5. Προτείνω σε κάθε περίπτωση να χρησιμοποιήσουμε `venv` (το εξηγώ [παρακάτω](#οδηγίες-για-venv)).

## Οδηγίες για vscode
 - <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>`</kbd>: Δημιουργία terminal που μπορεις να τρέξεις όλες τις εντολές απο τις οδηγίες.
 - Όταν ανοίγουμε κάποιο αρχείο, πάνω δεξιά βγάζει ένα κουμπάκι "play". Είναι ένας εναλλακτικός τρόπος να τρέξουμε το κάθε αρχείο.

## Οδηγίες για GitHub μέσα στο vscode
To vscode, στα αριστερά, στο tab για το source control (<kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>G</kbd>), έχει κάποιες διευκολύνσεις για το github:

- Πρώτη φορά - Κατέβασμα του project (Clone):
    1. Πατάμε <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>P</kbd> και γράφουμε **Git: Clone**.
    2. Επιλέγουμε **Clone from GitHub** και συνδεόμαστε με τον λογαριασμό μας αν μας ζητηθεί.
    3. Στο search box που εμφανίζεται, βρίσκουμε το repository της ομάδας και το επιλέγουμε. Αν δεν βλέπουμε κάποιο repository, μπορούμε απλά να κάνουμε copy paste το link του repository.
    4. Επιλέγουμε έναν φάκελο στον υπολογιστή μας όπου θα αποθηκευτεί το project και πατάμε **Select as Repository Destination**.
    5. Πατάμε **Open** στο μήνυμα που εμφανίζεται για να ανοίξουμε το project.

- Πώς δημιουργούμε καινούργιο Branch:
    1. Πατάμε τις 3 τελίτσες (`...`) πάνω δεξιά στο πάνελ του Source Control, δεξιά απο το `Changes` (πάμε το ποντίκι από πάνω για να εμφανιστούν).
    2. Επιλέγουμε **Branch** -> **Create Branch From...**
    3. Διαλέγουμε σε ποιο branch θα βασιστεί το καινούργιο μας (συνήθως το `main`).
    4. Γράφουμε το όνομα του νέου μας branch (π.χ. το όνομά μας ή το feature που θα φτιάξουμε) και πατάμε <kbd>Enter</kbd>.
    5. Στην κάτω μεριά του αριστερού πάνελ, εκεί που γράφει `Graph`, πατάμε το εικονίδιο που δείχνει βελάκι προς τα πάνω και συννεφάκι (Publish Branch).

- Πως αλλάζουμε branch:
    1. Στο πάνω μέρος του αριστερού πάνελ, εκεί που φτιάχνουμε καινούργιο branch, πατάμε **Checkout to...**
    2. Επιλέγουμε το branch στο οποίο θέλουμε να πάμε.

- Όταν αρχίζουμε την δουλειά (στο δικό μας branch):
    1. Στην κάτω μεριά του αριστερού πάνελ, εκεί που γράφει `Graph`, πατάμε στο `Auto` και επιλέγουμε το branch στο οποίο θα δουλέψουμε.
    2. Πατάμε τα 2 εικονίδια που βρίσκονται λίγο πιο δεξιά που δείχνουν τα βελάκια προς τα κάτω. Πρώτα αυτό που λέει **Fetch From All Remotes** και μετά το **Pull**. Έτσι, "τραβάμε" όποια αλλαγή μπορεί να έχει γίνει στον κώδικά μας από κάποιον άλλο. Ομοίως, με αυτόν τον τρόπο μπορεί ο καθένας να δουλέψει από όποιον υπολογιστή θέλει χωρίς να κάνει copy paste σε φλασάκια, drive, κτλ.

- Όταν τελειώνουμε την δουλειά:
    1. Αφού τελειώσουμε κάποια δουλειά, πάμε πάνω απο το αρχείο που πειράξαμε, πατάμε το `+` (`Stage Changes`) **Μεγάλη προσοχή! Αν πατήσουμε το `Discard All Changes` κακά πράγματα θα συμβούν (θα χαθούν όλες οι αλλαγές και δεν έχει επιστροφή. Όχι και πολύ καλή ιδέα να το βάλουν δίπλα απ' το `+` αν με ρωτάτε).**
    2. Αφού κάνουμε stage όλα τα αρχεία που χρειάζονται, γράφουμε ένα σύντομο μήνυμα στο κουτάκι λίγο πιο πάνω που γράφει `Message ...`. Δεν έχει και πάρα πολλή σημασία τι θα γράψουμε αλλά καλό είναι να περιγράφει εν συντομία τι πετύχαμε απ' το προηγούμενο commit μέχρι και τώρα.
    3. Πατάμε το μεγάλο κουμπί, `✓ Commit`
    4. Πατάμε το ίδιο κουμπί που τώρα γράφει `Sync Changes`.


## Οδηγίες για venv
Το venv (**v**irtual **env**ironment) μας χρησιμεύει στο να έχουμε όλοι ακριβώς το ίδιο περιβάλλον, π.χ. τις ίδιες βιβλιοθήκες. Μπορεί εγώ να έχω στον υπολογιστή μου κάποιο παλιότερο `tkinter` και κάτι που παίζει σε κάποιον άλλον να μην λειτουργεί σε εμένα, ή το αντίθετο.

Το αρχείο `requirements.txt` **πάντα** θα καταγράφει ακριβώς όλα όσα χρειάζεται το πρόγραμμά μας για να τρέξει σωστά.

Πιθανότατα δεν θα μας χρειαστεί ποτέ στην πραγματικότητα αλλά είναι best practice γενικά.

**Προσοχή: όλες τις εντολές τις τρέχουμε απο τον κεντρικό φάκελο του project μας.
Αν απλά ανοίξουμε ένα terminal με <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>`</kbd> τότε όλα καλά!**

### 1. Δημιουργία `venv`:
```bash
python -m venv .venv
```
### 2. Ενεργοποίηση `venv`:
- Windows (PowerShell):
```powershell
.\.venv\Scripts\Activate.ps1
```
- Windows (CMD) *(Λογικά δεν θα χρησιμεύσει σε κάποιον αυτό)*
```cmd
.\.venv\Scripts\activate.bat
```
- Linux / macOS:
```bash
source .venv/bin/activate
```

### 3. Εγκατάσταση απαιτήσεων (πιθανότατα θα είναι πάντα άδειο):
```bash
pip install -r requirements.txt
```
