Ο στόχος του project είναι να δημιουργηθεί σε ηλεκτρονική μορφή ένα απλό (επιτραπέζιο) παιχνίδι δύο παικτών, τη λίγο-πολύ γνωστή σε όλους Τρίλιζα (tic-tac-toe, στα αγγλικά). Βασικός σκοπός του παιχνιδιού είναι ο κάθε παίκτης να σχηματίσει τριάδα (τρίλιζα) με τα πούλια του, στα προσχεδιασμένα τετράγωνα (ταμπλό 9 θέσεων, σε διάταξη 3x3), ενώ ταυτόχρονα εμποδίζει τον αντίπαλο από το να πετύχει το ίδιο. Μία περιγραφή του παιχνιδιού δίνεται στην σχετική σελίδα της Wikipedia με τίτλο Tic-tac-toe: https://en.wikipedia.org/wiki/Tic-tac-toe).

Το παιχνίδι θα μπορεί να υποστηρίζει παίκτες που είναι φυσικά πρόσωπα (human players) αλλά και αυτοματοποιημένους παίκτες (computer player), και να παίζεται από συνδυασμούς παικτών (human to-human, human-to-computer, computer-to-computer).

Το παιχνίδι συστήνεται ισχυρά να υλοποιηθεί με γραφική διεπαφή, η οποία θα επιτρέπει την επιλογή των παικτών (από τις προαναφερόμενες κατηγορίες) και στη συνέχεια θα επιτρέπει την επαναληπτική διεξαγωγή διαδοχικών γύρων του παιχνιδιού με τους παίκτες να ξεκινούν εναλλάξ, ενώ θα παρουσιάζει και το έως κάθε στιγμή σκορ. Για τη γραφική διεπαφή συστήνεται να χρησιμοποιήσετε τη βιβλιοθήκη tkinter.

Για τους αυτοματοποιημένους παίκτες προεκτείνεται η υλοποίηση 
 - ενός παίκτη που παίζει εντελώς τυχαία σε κάποια ελεύθερη θέση (και προφανώς δεν παίζει και πολύ καλά)
 - ενός παίκτη που ακολουθεί μια απλοποιημένη στρατηγική (η οποία μπορεί να μην είναι και ιδιαίτερα καλή). Ενδεικτικά μια σχετικά καλή (αλλά όχι τέλεια) στρατηγική του παίκτη συνοψίζεται στους ακόλουθους κανόνες (με σειρά προτεραιότητας):
    1. συμπληρώνει τρίλιζα (εφόσον έχει αυτή τη δυνατότητα), ώστε να κερδίσει
    2. μπλοκάρει πιθανή τρίλιζα του αντιπάλου, εάν ο αντίπαλος μπορεί να κάνει τρίλιζα στην επόμενη κίνηση του, ώστε να αποτρέψει την άμεση ήττα του
    3. παίζει στην κεντρική θέση, εάν αυτή είναι ελεύθερη, 
    4. παίζει σε γωνιακή θέση, εάν κάποια από αυτές είναι ελεύθερη, 
    5. παίζει σε ελεύθερη θέση στο μέσο πλευράς

Για να υλοποιήσετε το παιχνίδι θα πρέπει να επιλέξετε την αναπαράσταση του ταμπλό, την αναπαράσταση των κατειλημμένων και κενών θέσεων, να σκεφτείτε πως θα εντοπίζετε τις επόμενες πιθανές κινήσεις, πως θα εντοπίζετε μια τρίλιζα (για να τερματίζεται ο γύρος του παιχνιδιού), πως θα εντοπίζετε μια κατάσταση του ταμπλό που μπορεί να δώσει τρίλιζα στο επόμενο βήμα και να συνδυάσετε τα παραπάνω για να υλοποιήσετε τους αυτοματοποιημένους παίκτες.

Επίσης θα πρέπει να σκεφτείτε τις δυνατές ενέργειες των χρηστών (επιλογή πρώτου και δεύτερου παίκτη, έναρξη του παιχνιδιού, τερματισμός του παιχνιδιού, επιλογή κίνησης κλπ.) και να σχεδιάσετε τον τρόπο παρουσίασης του ταμπλό και των επιλογών (π.χ. κουμπιά).

Για την υλοποίηση των παραπάνω συνίσταται να χρησιμοποιήσετε αντικειμενοστρεφή προγραμματισμό, αλλά αυτό δεν είναι απαίτηση του project.

Για την υλοποίηση μπορείτε να χρησιμοποιήσετε όποιες βιβλιοθήκες επιθυμείτε.

Μια υλοποίηση που καλύπτει τα παραπάνω θεωρείτε πλήρης για το σκοπό του project. Εάν όμως επιθυμείτε, μπορείτε να προσθέσετε επιπλέον λειτουργικότητα στην εφαρμογή σας.

Ενδεικτικές ιδέες για επιπλέον λειτουργικότητα της εφαρμογής (χωρίς συγκεκριμένη σειρά και με διαφορετικά επίπεδα δυσκολίας) είναι:
- Επιπλέον αυτοματοποιημένος παίκτης που παίζει τέλεια (π.χ. με αξιολόγηση όλων των δυνατών εξελίξεων του παιχνιδιού, με χρήση του αλγόριθμου min-max - https://en.wikipedia.org/wiki/Minimax)
- Υποστήριξη πολλών διαφορετικών παικτών που είναι φυσικά πρόσωπα (π.χ. διακρίνοντάς τους από το όνομα που θα δηλώνουν οι ίδιοι στην αρχή του παιχνιδιού)
- Καταγραφή των αποτελεσμάτων μεταξύ παικτών (πολλών διαφορετικών φυσικών προσώπων και των αυτοματοποιημένων παικτών) σε βάση δεδομένων – Παρουσίαση στατιστικών στοιχείων για κάποιο παίκτη ή για κάποιο ζεύγος παικτών – Παρουσίαση καλύτερων παικτών (top ten players)
- Υποστήριξη παιχνιδιού τριών παικτών σε ταμπλό 4x4
- Επιλογή διαφορετικών αναπαραστάσεων/εικόνων για τα πούλια των παικτών, ή/και άλλων γραφικών στοιχείων της εφαρμογής.
